# Schedule Fixes & Enhancements

**Phase:** 3b | **Version:** 0.19 | **Status:** Not Started

## Problem

The schedule has several reliability and UX issues that undermine trust in the scheduling engine. Tasks show as in-progress before the schedule has started. The in-progress indicator skips over breaks, making it look like you should be working when you're meant to be resting. Completing a task doesn't trigger a reschedule, so estimated finish times drift. The schedule page buttons also need visual attention.

## Requirements

### Core

- [ ] Fix: when a schedule is set to start in the future, the first task should not be marked as in-progress until the actual start time
- [ ] Add pause/play functionality:
  - Pause stops the schedule clock
  - Play reschedules all remaining tasks from the current time
  - No tracking of pause duration or reason (simple implementation)
- [ ] Auto-reschedule when a task is marked as done:
  - Remaining tasks shift to fill the gap
  - Estimated finish time updates accordingly
  - If the reschedule opens up space for another backlog task, prompt the user to add it
- [ ] Fix in-progress tag so it correctly handles breaks:
  - During a break period, no task should be marked as in-progress
  - The in-progress indicator should resume on the next task after the break ends

### UX

- [ ] Fix schedule page button styling — buttons look ugly and are poorly spaced from the header
- [ ] Pause/play should use clear, intuitive icons (pause ⏸ / play ▶)
- [ ] When prompting to add a task after reschedule, show the top-priority task that would fit in the available time
- [ ] Done items on the schedule should have their scheduled time removed or greyed out, as it may be out of sync after a reschedule

### Data Model

- [ ] Add schedule state: `paused` (boolean, default false)
- [ ] Store `pausedAt` timestamp when paused (cleared on resume)
- [ ] Ensure backward compatibility — existing schedules default to not paused

### Edge Cases

- [ ] Pause while on a break — resume should still reschedule from now
- [ ] All remaining tasks completed while paused — schedule should transition to finished state
- [ ] Auto-reschedule produces a schedule that extends past the original end time — show a warning to the user
- [ ] Last task marked done — no reschedule needed, transition to schedule complete

## Files Likely Affected

- `src/components/TaskSchedule.vue` — in-progress tracking, done item display, pause/play UI
- `src/composables/useTaskActions.js` — `getScheduleTasks()`, `getScheduleTimes()`, reschedule logic
- `src/views/ScheduleView.vue` — button styling, pause/play controls
- `src/stores/app.js` — schedule state (paused, pausedAt)

## Acceptance Criteria

1. [ ] First task is not in-progress when schedule start time is in the future
2. [ ] In-progress tag does not skip over breaks
3. [ ] Pausing stops the schedule; playing reschedules remaining tasks from now
4. [ ] Completing a task triggers auto-reschedule with updated finish time
5. [ ] If space opens after reschedule, user is prompted to add a task
6. [ ] Done items do not display stale scheduled times
7. [ ] Schedule page buttons are well-styled and properly spaced
