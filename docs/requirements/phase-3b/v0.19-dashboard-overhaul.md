# Dashboard Overhaul

**Phase:** 3b | **Version:** 0.19 | **Status:** Not Started

## Problem

The dashboard charts and stats are functional but feel disconnected from the app's sci-fi design language. The doughnut charts look clunky and out of place. The "completed" stat shows an all-time count which becomes meaningless as it grows. Stats are static text with no interactivity — users can't click through to explore the data behind the numbers.

## Requirements

### Core

- [ ] Replace doughnut charts with a visualisation style that fits the sci-fi aesthetic (e.g., bar charts, radial gauges, or custom styled components)
- [ ] Change the "completed" stat to show tasks completed in the last 7 days instead of all-time
- [ ] Make each stat card clickable, navigating to the relevant view or showing a more detailed breakdown

### UX

- [ ] Clickable stats should have a hover/active state indicating interactivity
- [ ] Navigation targets for clickable stats:
  - Completed (7 days) → Completed tab on Tasks view
  - Active tasks → Backlog tab on Tasks view
  - Priority breakdown → Backlog tab filtered by the clicked priority
  - Schedule stats → Schedule view
- [ ] New chart style should be responsive and readable on mobile
- [ ] Maintain the existing colour scheme for priority levels

### Edge Cases

- [ ] Handle zero state gracefully — empty charts and "0 completed this week" should still look good
- [ ] If the target view for a clickable stat requires a filter, pass it via route query params

## Files Likely Affected

- `src/views/DashboardView.vue` — chart components and stat cards
- `src/router/index.js` — route query params for filtered navigation
- `src/views/TaskView.vue` — accept and apply filter from route query params

## Acceptance Criteria

1. [ ] Doughnut charts are replaced with a visualisation that fits the sci-fi design
2. [ ] "Completed" stat shows tasks completed in the last 7 days
3. [ ] Clicking a stat navigates to the relevant view or shows a detailed breakdown
4. [ ] Charts and stats are responsive and readable on mobile
5. [ ] Zero/empty states display cleanly
