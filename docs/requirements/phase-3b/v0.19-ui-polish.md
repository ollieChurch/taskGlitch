# UI Polish & Responsive Fixes

**Phase:** 3b | **Version:** 0.19 | **Status:** Not Started

## Problem

Various UI elements across the app have rough edges that undermine the polished feel, especially on mobile. The profile page has not been updated to match the design overhaul from v0.14. Several small but noticeable issues (unnecessary scroll, squashed buttons, code-variable labels) erode the user experience.

## Requirements

### Layout & Scroll

- [ ] Fix task and backlog lists showing unnecessary scroll when all tasks fit on screen — lists should only scroll when content overflows
- [ ] Make "How it works" and "What's new" sections more discoverable — they are currently hidden below the fold with no indication there is more content to scroll to (add a scroll indicator, move above fold, or restructure the page)

### Profile Page

- [ ] Redesign profile page to match the current app style established in v0.14
- [ ] Fix logout button — currently ugly and squashed against the header, should have proper styling and spacing
- [ ] Replace code-variable labels on profile (e.g., `shortBreak`, `longBreak`) with human-readable labels (e.g., "Short Break", "Long Break")

### Modals

- [ ] Remove auto-focus on the name field when opening the edit task modal — it can be disruptive on mobile where the keyboard pops up immediately
- [ ] Add horizontal margin (left/right padding) to all modals on mobile viewports to prevent content touching the screen edge

### Edge Cases

- [ ] Profile label changes must not break Firebase data reads/writes — labels are display-only, keys remain unchanged
- [ ] Modal margin should only apply on small viewports; desktop modals should retain their current sizing

## Files Likely Affected

- `src/views/TaskView.vue` — list scroll behaviour
- `src/views/ProfileView.vue` — full redesign, labels, logout button
- `src/components/HeaderNav.vue` — logout button styling and spacing
- `src/components/TaskModal.vue` — remove auto-focus on name field
- `src/components/ui/BaseModal.vue` — horizontal margin on mobile
- `src/views/LoginView.vue` or landing page component — "How it works" / "What's new" discoverability

## Acceptance Criteria

1. [ ] Task and backlog lists do not scroll when all content fits on screen
2. [ ] Profile page matches the app's current design language
3. [ ] Logout button is properly styled with adequate spacing from the header
4. [ ] Profile labels are human-readable (not code variable names)
5. [ ] "How it works" and "What's new" are discoverable without guessing there is more content
6. [ ] Edit task modal does not auto-focus the name field
7. [ ] Modals have horizontal margin on mobile viewports
